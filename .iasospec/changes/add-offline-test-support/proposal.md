# Proposal: 添加离线测试能力支持

## 需求摘要

为 create-iaso-analysis CLI 添加离线测试能力，支持在没有网络环境下的测试场景，通过 `--offline` 参数使用本地模版目录替代 GitHub clone。

## 背景与动机

当前的脚手架工具需要从 GitHub clone 模版仓库，这导致：
1. 测试依赖网络环境，无法在离线环境下进行
2. 测试不稳定，受网络波动影响
3. 无法快速进行本地开发和调试

需要提供一种离线测试机制，让开发者可以在本地创建模拟模版仓库进行测试。

## 目标与成功标准

- CLI 支持 `--offline` 参数，使用本地模版目录
- 提供测试模版自动创建脚本（`test/` 目录）
- 测试目录自动加入 `.gitignore`
- 测试完成后可自动清理

**成功标准**：
- `npm run test:setup` 能自动创建测试模版
- `npx create-iaso-analysis --offline test-project` 能在离线环境下创建项目
- `npm run test:cleanup` 能清理测试产物

## 范围与边界

### In Scope（本次包含）

- `--offline` 命令行参数支持
- 本地模版目录识别和复制（跳过 git clone）
- 测试模版自动创建脚本
- `.gitignore` 更新（添加 `test/` 目录）
- npm scripts：`test:setup`、`test:cleanup`

### Out of Scope（本次不包含）

- 完整的 E2E 测试框架
- CI/CD 集成
- 多种模版场景测试

## 用户/系统场景

### 场景 1：离线环境测试

- **谁**：开发者
- **何时/条件**：需要在无网络环境下测试脚手架功能
- **做什么**：
  1. 运行 `npm run test:setup` 创建本地测试模版
  2. 运行 `npx create-iaso-analysis --offline my-project`
- **得到什么**：使用本地模版创建的项目，无需网络连接

### 场景 2：本地开发调试

- **谁**：开发者
- **何时/条件**：开发脚手架新功能时需要快速验证
- **做什么**：修改代码后直接用 `--offline` 测试
- **得到什么**：快速反馈循环，无需等待网络请求

## 约束与假设

### 约束

- `--offline` 模式下，模版路径必须是有效的本地目录
- 测试模版结构需要模拟真实模版的最小功能集

### 假设

- 测试目录 `test/` 不需要提交到 Git
- 用户已安装必要的包管理器用于测试

## 名词与术语
